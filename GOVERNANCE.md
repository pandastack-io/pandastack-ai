# Governance

PandaStack uses a pragmatic BDFL model while the project is young.

Ajay Kumar, as founder and BDFL, has final decision authority on project direction, releases, maintainership, and disputes. Day-to-day decisions should happen in issues and pull requests using lazy consensus: if a proposal is clear, scoped, and no maintainer objects after reasonable review time, it can move forward.

Disagreements should be resolved with technical evidence, prototypes, and user impact. If consensus is not reached, the BDFL decides and documents the reasoning.

After 1.0, we plan to form a steering committee and delegate ownership for major areas such as agent, API, dashboard, SDKs, docs, and infrastructure examples.
